URL

Une URL, pour Uniform ressource locator, est une chaine de caractères uniforme qui permet d’identifier une ressource sur le world wide web.



Le format d’une URL est standardisé. Elle commence toujours par le protocole utilisé suivi de « : ». Ces protocoles sont par exemple :

  • http, https, -> hypertext transfer protocol (secure) pour accéder aux page web
  • ftp, -> file transfer protocol, pour accéder serveur
  • mailto, -> lien direct email
  • file, -> lien direct pour accéder un fichier
  • ip -> lien direct pour accéder un serveur
  • irc -> lien à une chatroom
  • torrent -> lien standardisé pour partager des fichiers

La partie suivante de l’URL représente l’hôte, identifié par son nom de domaine ou son adresse IP. S’il est omis, l’url désigne alors un emplacement sur la machine utilisée et pas une ressource externe.

Le nom de domaine peut être préfixé d’un nom d’utilisateur et d’un mot de passe séparé du nom de domaine par une arobase. Ce qui peut servir à limiter certains accès.

Le nom de domaine peut aussi être suffixé par le port précis à utiliser pour se connecter à la machine où la ressource est située.

Ensuite il y a le chemin à utiliser pour trouver la ressource. Ce chemin peut-être le chemin absolu pour trouver la ressource ou un chemin défini par le serveur contacté.

Pour finir viennent les données complémentaires, il en existe deux types.

D’abord les requêtes, préfixées d’un point d’interrogation transmettent des données complémentaires au serveur. Comme le contenu d’un champ rempli.

Ensuite les ancrages, préfixés d’un dièse, précisent un emplacement sur la page. Lors du chargement de la page, le navigateur va afficher directement l’emplacement précisé et pas le début de la page.
